# AI Image Direct Prompt v7

Reference-adaptive safe-area campaign system.

Generic improvement from v6:
- Do not force a fixed palette, product motif, or app category.
- Infer palette, illustration/rendering style, typography feel, background language, depth, shadows, and decorative density from IMAGE 1.
- Keep every feature screen screenshot-first and phone-first.
- Add strict safe-area requirements so the full phone frame remains visible and not cropped.
- Ask for the phone to be 8–12% smaller if needed to preserve bottom, side, and top margins.
- Keep cover typography and hero object inside intentional safe areas.
- Forbid generic artifacts across categories: fake UI cards, labels, speech bubbles, signs, numbers, AI chips, stickers, badges, fake claims, and readable decorative text.
- Allow characters/objects only when the reference style and product category clearly support them, and never over the phone/UI.

Reason:
- v6 improved coherence and screenshot preservation, but it was too specific to one app/reference style.
- v7 turns the winning idea into a reusable prompt principle: reference-adaptive campaign consistency + strict safe areas.

Iteration rule:
- Alternate between available references each iteration to avoid overfitting to one reference.
- v7 should be tested with `reference_02` because v6 used `reference_01`.

Success criteria:
- Full phone frame visible in every feature image.
- Same campaign system across cover and features.
- Style follows the chosen reference, not hardcoded colors/motifs.
- Real app UI remains readable and unmodified.
- No fake readable text outside the real screenshot.
