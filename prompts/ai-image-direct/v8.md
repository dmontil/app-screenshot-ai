# AI Image Direct Prompt v8

Creative-quality first, split cover/feature contract.

Generic direction change from v7:
- Stop optimizing only for absence of mistakes.
- Optimize for top-tier App Store creative quality: emotional hook, reference match, composition depth, screenshot fidelity, and campaign consistency.
- Separate cover and feature rules.

Cover contract:
- Cover is a marketing hero, not a screenshot-preservation screen.
- If the reference is character-led, playful, mascot-led, hand-led, object-led, or highly composited, create an app-specific buddy/hero/composite in that spirit.
- If the reference is minimal/product-led, keep the cover cleaner and product-symbol-led.
- App screenshot/device can be composited into the cover if it improves marketing impact.
- Avoid fake claims, fake awards, fake ratings, fake badges, and readable decorative text not provided by input/screenshot.
- Do not make the cover sterile just to be safe.

Feature contract:
- Feature screens remain screenshot-first.
- Real UI must stay readable and mostly unchanged.
- Buddy/object/character may appear only if the reference clearly supports it, and must stay outside the phone/UI.
- Preserve campaign energy and composition depth instead of falling back to empty backgrounds.
- Keep typography treatment consistent across the pack unless the reference varies it.

Iteration rule:
- Alternate references every iteration.
- v8 should be tested with `reference_01` because v7 used `reference_02`.

Success criteria:
- Cover regains emotional marketing quality comparable to the v1/v2 baseline.
- Features remain readable while feeling designed, not sterile.
- Prompt remains generic across app categories and references.
