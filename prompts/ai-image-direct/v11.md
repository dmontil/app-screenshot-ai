# AI Image Direct Prompt v11

Set-level panoramic campaign system.

Generic direction:
- Optimize the complete 5-image App Store set, not each image independently.
- Treat the pack as a shared horizontal campaign world/panorama sliced into vertical screenshots.
- Every screen should share background system: palette, lighting, horizon/ground/sky or abstract field, depth, decorative vocabulary, and typography treatment.
- Add screen index awareness (`screen X of N`) so each generation knows its role in the set.
- Allow subtle mockup variation across feature screens: angle, rotation, scale, or perspective, as long as UI remains readable.

Cover contract:
- Screen 1 is the hero segment.
- Keep app-specific buddy/hero rule: borrow reference style, never copy literal mascot identity.
- Include product proof: composited app screenshot/device when useful.
- Realistic App Store badge allowed on cover.
- Establish the background world that continues through screens 2-N.

Feature contract:
- Screens 2-N are screenshot-first.
- Use the same background world, not new unrelated scenes.
- Visual energy should come from shared background/side areas.
- Hard moat around phone/UI: nothing overlaps screenshot or touches phone frame.
- Vary mockup angle/position subtly for vitality.
- No readable fake text outside screenshot.

Iteration rule:
- v11 uses `reference_02` because v10 used `reference_01`.
- v11 should be tested with `count=5`.

Success criteria:
- The five outputs feel like one campaign set.
- Backgrounds are visibly related, almost slices of one wider world.
- Mockups have enough variation to avoid template fatigue.
- Cover has hook + product proof.
- Features preserve UI readability.
