# AI Image Direct Prompt v10

App-specific buddy + feature-safe campaign energy.

Generic direction from v9:
- Keep cover creative quality and marketing impact.
- If the reference has a mascot/buddy, borrow its visual language but never copy its literal identity/species/object.
- Create a buddy/hero that belongs to the app category and value proposition.
- Keep realistic App Store badge allowed on cover if it looks official and crisp.
- Tighten feature screens: no foreground buddy/object by default; visual energy must come from background/side design with a clear moat around the phone.

Cover contract:
- IMAGE 1 is art director for style only: palette, rendering, shape language, emotion, depth, typography feel.
- Do not copy reference mascot/species/identity/pose.
- If character-led, create a new app-specific buddy/hero.
- If minimal/product-led, use product symbol/device/abstract hero.
- Device/screenshot may be composited for marketing impact.
- Realistic App Store badge allowed.
- No cheap fake text/cards/stats/claims unless sourced from input/screenshot.

Feature contract:
- Screenshot is hero.
- No dominant foreground buddy/object/card/sticker/panel by default.
- Use background/side areas for energy: shapes, hills, clouds, rays, gradients, patterns, atmosphere.
- Hard moat around phone: nothing overlaps or touches phone/UI.
- No readable fake text outside screenshot.

Iteration rule:
- v10 uses `reference_01` because v9 used `reference_02`.

Success criteria:
- Cover creates an app-specific hero instead of copying reference mascot.
- Feature screens preserve UI while keeping visual energy.
- Prompt remains generic across app categories and references.
