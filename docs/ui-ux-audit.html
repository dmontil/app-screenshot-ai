Hallazgos clave

1
El primer pliegue no muestra el producto.

No hay ejemplo visual del output, mockup de screenshot App Store, ni flujo resumido.

2
La app parece un panel interno.

“Provider settings”, JSON reports y modelos aparecen antes que el valor para el usuario.

3
Carga cognitiva alta.

Se piden API keys, metadata, labels y categorías antes de sentir progreso.

4
Acciones primarias poco diferenciadas.

Generar, cargar, refrescar y descargar comparten estilo; falta un CTA dominante por estado.

5
No hay modo “simple”.

Usuarios nuevos deberían poder usar Fixture/demo con datos precargados y entender el resultado en 30 segundos.

6
Reportes técnicos sin traducción.

Quality report y storyboard crudos deberían plegarse detrás de “Advanced / Developer”.

Dirección de producto recomendada

Antes: formulario pipeline

  • Hero textual.
  • Provider primero.
  • Secciones numeradas largas.
  • Previews aparecen solo tras generar.
  • JSON visible como parte normal del flujo.

Después: estudio de screenshots

  • Hero con preview del resultado.
  • Stepper: Upload → Brand → Generate → Edit → Export.
  • Defaults inteligentes y modo demo.
  • Canvas visual + panel de propiedades.
  • Advanced plegado para API/model/reportes.

Sistema visual recomendado

Warm Pro

One CTA

Un botón principal por estado: Try demo, Generate, Save, Download.

Progress

Mostrar progreso por pasos, validaciones y “next best action”.

Trust

Local-first y BYOK como garantías, no como primer trabajo del usuario.
